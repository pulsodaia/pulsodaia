Claude Code Atualiza Configurações e Integrações

Ferramenta de desenvolvimento com IA foca em persistência e flexibilidade para usuários brasileiros.

Por Pulso da IA · 23 de abr de 2026 · 3 min de leitura
Imagem: github.com

A versão v2.1.119 do Claude Code introduz melhorias significativas na gestão de configurações, permitindo que ajustes de tema, modo de editor e verbosidade sejam salvos localmente. A atualização também expande a compatibilidade com sistemas de controle de versão, tornando a ferramenta mais adaptável a fluxos de trabalho diversos.

O que aconteceu

A mais recente atualização do Claude Code, a versão v2.1.119, traz um conjunto de aprimoramentos focados em usabilidade e integração. Uma das mudanças centrais é a persistência das configurações. Agora, as preferências de tema, modo de editor, verbosidade e outros ajustes são salvos no arquivo ~/.claude/settings.json. Isso significa que as personalizações feitas pelo usuário serão mantidas entre as sessões e poderão ser sobrescritas por configurações de projeto ou locais, seguindo uma ordem de precedência clara.

Adicionalmente, foi introduzida a configuração prUrlTemplate. Essa opção permite que o link de "PR badge" no rodapé aponte para URLs de revisão de código customizadas, em vez de direcionar sempre para o GitHub.com. Outra novidade é a variável de ambiente CLAUDE_CODE_HIDE_CWD, que possibilita ocultar o diretório de trabalho atual na tela de inicialização do programa.

A funcionalidade --from-pr foi ampliada para aceitar URLs de merge requests do GitLab e pull requests do Bitbucket, além dos já suportados do GitHub Enterprise. Por fim, o modo --print agora respeita as diretivas tools: e disallowedTools: definidas no frontmatter do agente, replicando o comportamento observado no modo interativo. A capacidade de invocar agentes específicos via --agent também foi refinada para honrar as configurações de ferramentas. Detalhes completos das alterações podem ser encontrados no repositório oficial.

Por que importa

Para desenvolvedores que utilizam o Claude Code no Brasil, essas atualizações representam um avanço na personalização e na eficiência do fluxo de trabalho. A persistência das configurações reduz a necessidade de reconfigurar a ferramenta repetidamente, economizando tempo em cada nova sessão de desenvolvimento. A capacidade de integrar-se a diferentes plataformas de controle de versão, como GitLab e Bitbucket, amplia o leque de projetos onde o Claude Code pode ser aplicado sem adaptações complexas.

A flexibilidade de apontar o "PR badge" para sistemas de revisão de código internos ou específicos da empresa também é um benefício importante para organizações com fluxos de trabalho de code review particulares. A ocultação do diretório de trabalho pode ser útil em ambientes onde a discrição sobre a estrutura de pastas é desejada. Ao alinhar o comportamento dos modos --print e interativo em relação ao uso de ferramentas, a consistência na execução de agentes é garantida, minimizando surpresas e erros.

O que esperar

Com essas melhorias, o Claude Code consolida sua posição como uma ferramenta de desenvolvimento assistido por IA mais robusta e adaptável. A tendência é que futuras atualizações continuem a refinar a integração com sistemas de desenvolvimento e a oferecer maior controle sobre o comportamento do agente. A expansão do suporte a diferentes provedores de controle de versão sugere um caminho de maior interoperabilidade. A comunidade de desenvolvedores pode esperar um Claude Code cada vez mais alinhado às diversas necessidades de um ambiente de desenvolvimento moderno. Para mais informações sobre as funcionalidades e o futuro da ferramenta, consulte o changelog detalhado.
